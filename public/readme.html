Store Information

Store Name: Furry Friends Pet Store

What we're selling: We specialize in selling live pets including dogs, cats, parrots, and rabbits. Each pet comes with health certificates, vaccination records, and a care package with essential supplies.

Additional Pages Implemented

Beyond the required functionality, I implemented the following additional pages:

1. Pet Information Page (pet-info.html)

Purpose: Detailed information about each individual pet including comprehensive care instructions.

How to operate: Click "Pet Info" button on any pet in the store or category pages. Shows detailed care information for diet, housing, health, exercise, grooming, and social needs specific to the pet type.

2. Category Pages (category.html)

Purpose: Dedicated pages for each pet category (dogs, cats, parrots, rabbits) with category-specific information.

How to operate: Click on category cards from the home page or use category filters in the store. Each category page shows only pets of that type with relevant category description.

3. Thank You Page (thank-you.html)

Purpose: Post-purchase confirmation page with order details and next steps.

How to operate: Automatically redirected here after successful payment. Shows purchase ID, total amount, and information about delivery and care packages.

4. Registration Page (register.html)

Purpose: User registration with form validation and error handling.

How to operate: Fill out username (min 3 chars), email, and password (min 6 chars). Includes password confirmation and redirects to login after successful registration.

Challenges & Difficulties

The most challenging aspects of this project were:

  • Cookie Management: Implementing proper cookie expiration for "remember me" functionality (12 days vs 30 minutes) and ensuring cookies work correctly across all pages.
  • File Upload Handling: Setting up multer for image uploads in the admin panel while also supporting URL-based images as fallback.
  • Data Persistence: Designing a file-based storage system that maintains data integrity across server restarts without using a traditional database.
  • DOS Protection: Implementing rate limiting middleware that tracks requests per IP address and resets counters appropriately.
  • Error Handling: Creating comprehensive async error handling throughout all modules while maintaining user-friendly error messages.

Routes Supported

Static Routes:

  • GET / - Home page (index.html)
  • GET /readme.html - This README page
  • GET /llm.html - LLM usage documentation

Authentication Routes:

  • POST /api/register - User registration
  • POST /api/login - User login with remember me option
  • POST /api/logout - User logout
  • GET /api/check-auth - Check authentication status

Store Routes:

  • GET /api/products - Get all products (with optional search)
  • GET /api/products/category/:category - Get products by category
  • GET /api/products/:id - Get single product details

Cart Routes:

  • GET /api/cart - Get user's cart contents
  • POST /api/cart - Add item to cart
  • DELETE /api/cart/:productId - Remove item from cart

Checkout Routes:

  • POST /api/checkout - Process payment and complete purchase
  • GET /api/purchases - Get user's purchase history

Admin Routes:

  • GET /api/admin/activities - Get user activity logs (with optional filter)
  • POST /api/admin/products - Add new product (admin only)
  • DELETE /api/admin/products/:id - Delete product (admin only)

Additional Routes:

  • GET /api/pet-info/:category - Get pet care information by category

Testing Approach

The application was tested using the following methods:

  • Automated Testing: test.js file contains comprehensive route testing using node-fetch to verify all API endpoints
  • Manual Testing: Each user flow was tested manually including registration, login, shopping, cart management, checkout, and admin functions
  • Error Testing: Tested various error conditions including invalid logins, empty carts, unauthorized access, and network failures
  • Cross-browser Testing: Verified functionality across Chrome, Firefox, and Safari
  • Cookie Testing: Verified "remember me" functionality with different expiration times
  • File Upload Testing: Tested image uploads and URL-based images in admin panel
  • DOS Protection Testing: Verified rate limiting works correctly under high request volumes
